# منصة دواير / مسافتي — ملف المشروع الكامل (Project Spec)

> **الفكرة الأساسية:** بدل ما الناس تفهم نفسها بالكلام… تفهم نفسها **بالمكان**.  
> كل إنسان = نقطة — كل علاقة = موقع — كل حد = مسافة — كل وجع = اختراق دائرة.

---

## 0) أسماء المشروع (Brand Architecture)

- **اسم المنتج (MVP):** دواير  
- **الاسم الداخلي/الطبقة:** مسافتي (Feature/Module أو اسم المنتج داخل منظومة “رحلتي” لاحقًا)  
- **التموضع:** أداة قرار/وعي تطبيقي — مش علاج نفسي، ومش وعظ، ومش تشخيص.

**Tagline (اقتراح أساسي):**  
- شوف حدودك… وقّف الاستنزاف.  
- كل علاقة ليها مكان.

---

## 1) المشكلة (Problem)

المستخدم غالبًا:
- فاهم نفسيًا كتير (قراءة/محتوى/نصايح)  
- بس مش قادر يطبّق  
- عايش في: تحليل زيادة + ذنب + استنزاف علاقات  
- محتاج “يشوف” نفسه بدل ما يسمع شرح جديد

**الحقيقة المؤلمة:**  
- الكلام لوحده مش بيغيّر سلوك  
- النصايح ساعات بتزوّد ذنب  
- التحليل بيسرق الحضور  
- الوعي محتاج يتجسّد

---

## 2) الحل (Solution)

منصّة “دواير/مسافتي” بتحوّل الوعي إلى **فعل** عبر:
- **خريطة علاقات بصرية** (Drag & Drop)
- **دوائر قرب/مسافة** بألوان واضحة
- **طبقة أسئلة إحساس قصيرة** (AI Layer) بدون تشخيص
- **Action Toolkit**: بروتوكولات خروج وحدود وجُمل جاهزة
- **زر طوارئ** ثابت وقت التوتر/الدخول في دور غلط

---

## 3) فلسفة المنتج (Product Philosophy)

### المبادئ
- **لا تشخيص.**  
- **لا تصنيفات مرضية.**  
- **لا وعظ.**  
- **الجسم هو المرجع** مش النظرية.  
- **التوصيف بدون حكم.**  
- **الحدود = وعي مش قسوة.**

### لماذا مختلفون؟
معظم الحلول: تشرح/تعالج/تشخّص  
**دواير:** بتخلّيك *تشوف* وتاخد قرار بسرعة

---

## 4) النموذج الذهني الأساسي (Core Model)

### 4.1 الدائرة المركزية — “أنا المركز” (ثابتة لا تتحرك)
- الجسد  
- الوقت  
- الطاقة  
- الفلوس  
- المساحة النفسية

### 4.2 الدواير حول المركز
- 🟢 **الدائرة الأولى — القُرب الصحي**
  - ناس آمنة، راحة، مفيش مجهود، مفيش تحليل
- 🟡 **الدائرة الثانية — القرب المشروط**
  - قرايب، شغل، علاقات فيها تاريخ… محتاجة حدود
- 🔴 **الدائرة الثالثة — الخطر/الاستنزاف**
  - ذنب، لعب أدوار، تحميل، لوم، سحب طاقة

> **قاعدة تشغيل:** مش كل قرب صح… ومش كل بعد قسوة.

---

## 5) الجمهور المستهدف (Target Users)

- ناس “شايلة كتير”  
- ناس تعبانة من التحليل  
- ناس عندها وعي عالي + إرهاق عصبي  
- “طيبة زيادة عن اللزوم”  
- كوتشز/ثيرابيست/صنّاع محتوى (كأداة مساعدة لعملائهم)

---

## 6) رحلة المستخدم (End-to-End User Journey)

### المرحلة 0 — قبل الدخول
- إحساس: “فاهم… بس مش عارف أطبّق”

### المرحلة 1 — Landing → Onboarding
- رسالة واحدة: **“مش محتاج تفهم أكتر… محتاج تشوف نفسك.”**
- زر: **ابدأ الخريطة**
- تسجيل مش إجباري في البداية

### المرحلة 2 — Goal Picker (سؤال واحد)
“إيه أكتر حاجة مرهقاك دلوقتي؟”
- علاقات قريبة / العيلة / الحب / الشغل / الفلوس والذنب / مش عارف

### المرحلة 3 — بناء أول خريطة (Core Moment)
- شاشة “أنا المركز”
- إضافة أشخاص كنِقَط Drag & Drop

### المرحلة 4 — Insight (AI Prompt)
بعد 3–5 أشخاص:
- “في شخص قريب منك… بس جسمك مش مرتاح.”

### المرحلة 5 — Feeling Check (3 أسئلة)
Yes/No:
- بعد التعامل… بتتقل؟
- بتفكر فيه أكتر مما تحب؟
- بتحس بذنب لو بعدت؟

### المرحلة 6 — Action Toolkit
حسب الدائرة:
- 🔴: جملة خروج + خطة تقليل تدريجي + زر “مش دوري”
- 🟡: حدود ناعمة + جمل قصيرة + توقيت تعامل
- 🟢: رسالة تطمين وتوسيع آمن

### المرحلة 7 — زر الطوارئ (Daily Use)
زر ثابت: **🛑 داخل دور غلط**
- جملة فصل
- حركة جسدية
- خروج هادي

### المرحلة 8 — التوسع
Maps Hub: عيلة/حب/شغل/فلوس… تفتحها لما تكون جاهز

### المرحلة 9 — التتبّع الهادئ
- عدد مرات “خرجت بدري”
- علاقات اتحركت
- رسالة أسبوعية بسيطة

---

## 7) أنواع الخرائط داخل المنصّة (Maps)

### 7.1 خريطة العيلة
- أم/أب/أخ/أخت… مع بروتوكولات خاصة لكل دور

### 7.2 خريطة الحب
- قرب صحي × تعلق مؤلم  
- مؤشرات: قلق/تحليل/ذوبان

### 7.3 خريطة الشغل
- مدير/زميل/عميل/شريك  
- قواعد: مين تفتح له؟ مين كلام رسمي؟ مين كل حاجة مكتوبة؟

### 7.4 خريطة الفلوس والذنب
- مين بياخد من طاقتك؟  
- صرف بقاء × صرف ذنب  
- حدود مالية مرئية

---

## 8) المواصفات الوظيفية (Functional Specs)

### 8.1 Core Map (Canvas)
- مساحة رسم مركزية (Canvas)
- دوائر متعددة حول المركز
- **نِقاط** تمثل أشخاص/علاقات
- Drag & Drop بين الدواير
- لون النقطة يتغير حسب الدائرة
- On Drop: اهتزاز خفيف (Haptic) + Toast قصير

### 8.2 إضافة شخص
- Modal / Bottom sheet
- اسم/لقب
- أيقونة اختيارية
- إنشاء Node جديدة

### 8.3 Insight Prompt
- يظهر بعد Threshold (مثلاً 3–5 نقاط)
- يرشّح “نقطة” عليها توتر/تناقض (قريب مكانًا لكن إجابات الإحساس تقول غير كده)

### 8.4 Feeling Check
- 3 أسئلة فقط
- نتائج: “العلاقة دي محتاجة حد” + نوع (قريبة/مشروطة/مرهِقة)
- **ده توصيف… مش حكم**

### 8.5 Action Toolkit
- **Copy Button** لجُمل جاهزة
- بروتوكول تقليل تدريجي
- زر: “🛑 مش دوري”
- محتوى قصير جدًا (Actionable)

### 8.6 Emergency Button (زر الطوارئ)
- ثابت Floating
- شاشة طوارئ:
  - جملة فصل قصيرة
  - حركة جسدية بسيطة
  - زر “خروج هادي”

### 8.7 Maps Hub
- شبكة خرائط
- تفعيل “افتح خريطة لما تحس إنك جاهز”

### 8.8 Progress / Tracking
- بدون أرقام معقدة
- مؤشرات بسيطة
- رسالة أسبوعية

### 8.9 Profile / Settings
- تغيير ألوان/لغة
- تصدير الخريطة (صورة/PDF)
- إعدادات الخصوصية

## 10) Copy & Tone (دستور النبرة)

### 10.1 شخصية الصوت
- هادي — مباشر — حاضر  
- مش دكتور/مش شيخ/مش منقذ  
- “واقف جنبك” مش “فوقك”

### 10.2 قواعد صارمة
- جمل قصيرة (سطر أو سطرين)  
- **لا**: لازم/المفروض/صح وغلط/تشخيص/اضطراب/سام/شفاء/طفولة/استحقاق  
- لا نصايح مباشرة، لا شرح نظري  
- نبرة الطوارئ = أوامر قصيرة (وقف/اخرج)

### 10.3 أمثلة Copy (مختصر)
- “وقف.”  
- “مش دورك دلوقتي.”  
- “القرب اختيار.”  
- “العلاقة دي مُرهِقة. ده توصيف… مش حكم.”

## 14) المتطلبات التقنية (Tech Notes)

> بما إنك بتشتغل على Cursor: خليك على MVP سريع + قابل للتوسّع.

### 14.1 Architecture (اقتراح عملي)
- Frontend: React + Canvas/SVG (حسب راحتك)
- State: Zustand أو Redux Toolkit (أو React Context بسيط في MVP)
- Backend: Firebase/Supabase/Node (اختيار حسب سرعتك)
- Auth: Email/Google (وبعدها SSO مع رحلتي)
- Storage: Cloud Storage (للتصدير لاحقًا)

### 14.2 Data Model (أبسط شكل)
**User**
- id, name, settings

**Map**
- id, userId, type (relations/family/love/work/money), createdAt

**Node (Person/Relation)**
- id, mapId, label, icon, ring (green/yellow/red), x,y

**InsightAnswers**
- nodeId, q1,q2,q3, createdAt

**ActionsLog**
- userId, nodeId, actionType (exit/copy/protocol), createdAt

### 14.3 Privacy
- Default: Private by design
- Export اختياري
- لا مشاركة اجتماعية تلقائية
- لا “تشخيص” ولا “حفظ بيانات حساسة” خارج الضروري
