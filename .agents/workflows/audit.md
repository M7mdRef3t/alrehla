# Audit Workflow

For any task in this workspace, do a strict pre-edit audit before making changes.

## Steps
1. Restate the task in one clear sentence
2. Identify the affected areas of the repository
3. Mention likely impact on:
   - routes
   - services
   - state
   - analytics
   - auth
   - database
   - deployment
4. Choose the most relevant available skills and use them
5. Summarize:
   - facts
   - risks
   - hidden risks
   - regression risks
   - missing information
6. Propose the smallest safe next step
7. Do not start broad implementation before the audit is complete

## Output format
- Task
- Affected areas
- Skills used
- Facts
- Risks
- Hidden risks
- Regression risks
- Smallest safe next step
