# ملخص التحديث: الصياغة الشخصية + الأعراض 🎯

## التغييرات الرئيسية

### 1. ✅ الصياغة الشخصية (Personalized Messaging)

#### قبل:
```
قرب مؤلم محتاج حماية
الشخص: ماما — الدائرة: استنزاف

القرب هنا بيضغط على جسمك أكتر مما بيطمنه.
```

#### بعد:
```
قربك من "ماما" مؤلم ومحتاج حماية
الحالة: استنزاف

فهم الوضع:
علاقتك مع "ماما" بتاخد منك أكتر مما بتديك. جسمك بيحذرك - اسمع له.
```

**الفرق:**
- العنوان بقى **شخصي** - "قربك من ماما" بدل "قرب مؤلم"
- استبدلنا "الدائرة" بـ "الحالة" (أوضح للمستخدم)
- الرسالة في "فهم الوضع" بتستخدم اسم الشخص مباشرة

---

### 2. ✅ Symptoms Checklist (قائمة الأعراض)

تم إضافة قائمة أعراض تفاعلية بعد "فهم الوضع" مباشرة:

#### العناصر:
- **3 فئات**: عاطفي 💭 | جسدي 🫀 | سلوكي 🎯
- **Checkboxes تفاعلية** مع animations
- **عداد الأعراض المختارة**: "اخترت X عرض"
- **حفظ تلقائي** في الـ State

#### الأعراض حسب الدائرة:

**الدائرة الحمراء (استنزاف) - 9 أعراض:**
- بتحس بالذنب لما تقول 'لأ'
- بترجع من اللقاء منهك نفسياً
- بتفكر في الكلام لساعات بعد اللقاء
- بتحس إنك 'مش كفاية' مهما عملت
- بتتجنب المكالمات أو اللقاءات
- بتنسى احتياجاتك عشان ترضيه/ترضيها
- بتحس بتوتر جسدي (صداع، ألم معدة)
- بتحس إنه/إنها بتلاعبك عاطفياً
- بتنسى مين أنت أو إيه اللي بتحبه

**الدائرة الصفراء (محتاجة انتباه) - 6 أعراض:**
- العلاقة مرات حلوة ومرات مرهقة
- بتحسب كلامك عشان ما يزعلش
- بتحس بالقبول بس لما تعمل اللي هو/هي عايزه
- مش متأكد لو العلاقة دي صحية ولا لأ
- بتحاول ترضيه/ترضيها على حساب راحتك
- مش عارف إيه الحدود المناسبة

**الدائرة الخضراء (صحية) - 6 أعراض:**
- بتحس بالأمان وانت معاه/معاها
- بترجع من اللقاء مبسوط ومرتاح
- بتقدر تكون نفسك من غير تصنع
- في احترام متبادل وتقدير
- الحدود واضحة ومحترمة من الطرفين
- العلاقة متوازنة - بياخد ويدي

## 🎯 الفوائد

### 1. Personalization أعلى
- المستخدم بيحس إن الرسالة **ليه هو شخصياً**
- استخدام اسم الشخص في كل مكان بيزود الـ emotional connection

### 2. Self-Awareness (الوعي الذاتي)
- قائمة الأعراض بتخلي المستخدم يفكر: "فعلاً أنا بحس بكده!"
- بتساعده يوضّح مشاعره الغامضة

### 3. Data for Better Plans
- الأعراض اللي المستخدم بيختارها هتُستخدم في:
  - توليد خطة تعافي أدق
  - تخصيص التمارين والتدريبات
  - تحديد الأولويات في الخطة

### 4. Progress Tracking
- نقدر نتتبع: هل الأعراض قلّت بعد التدريب؟
- نقدر نطلب من المستخدم يراجع الأعراض كل أسبوع

```typescript
const selectedSymptoms = node.analysis.selectedSymptoms;

if (selectedSymptoms.includes('guilt')) {
  // Add specific exercises for guilt
  plan.addExercise({
    title: "تمرين: تحدي الذنب المفتعل",
    description: "..."
  });
}

if (selectedSymptoms.includes('exhausted')) {
  // Add energy management strategies
  plan.addExercise({
    title: "استراتيجيات حماية الطاقة",
    description: "..."
  });
}
```

### مقترحات إضافية:

1. **شاشة "مراجعة الأعراض"** (بعد شهر):
   - "راجع الأعراض اللي كنت بتحس بيها قبل كده"
   - "شيّك على اللي لسه موجود"
   - Compare: Before vs After

2. **Symptom-Specific Advice**:
   - كل عرض يكون ليه نصيحة مخصصة
   - مثلاً: "بتحس بالذنب؟ جرب تقنية X"

3. **Severity Scale** (مستقبلاً):
   - بدل checkbox بسيطة، نضيف مقياس:
   - ☐ ما بحسش بيه
   - ☐ أحياناً
   - ☐ غالباً
   - ☐ دايماً
