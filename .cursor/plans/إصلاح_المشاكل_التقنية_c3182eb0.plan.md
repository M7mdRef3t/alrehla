# خطة إصلاح المشاكل التقنية

## المشاكل المطلوب إصلاحها

### 1. إضافة ملف `netlify.toml` للنشر السليم

**المشكلة**: المشروع لا يحتوي على ملف تكوين Netlify، مما قد يسبب مشاكل في:

- عدم إعادة توجيه المسارات لـ React Router بشكل صحيح
- عدم تحديد مجلد البناء والأمر الصحيح

**الحل**:

- إنشاء ملف `netlify.toml` في جذر المشروع
- تحديد مجلد البناء `dist` وأمر البناء `npm run build`
- إضافة redirect rules لدعم SPA routing

```toml
[build]
  publish = "dist"
  command = "npm run build"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

### 2. إصلاح خطأ TypeScript في `mapState.ts`

**المشكلة**: في `[src/state/mapState.ts](src/state/mapState.ts)`:

- الدالة `markTrainingComplete` موجودة في الـ store (سطر 331-339)
- لكنها غير معرّفة في الـ interface `MapState` (سطر 5-25)

هذا يسبب:

- خطأ TypeScript عند الاستخدام
- عدم ظهور الدالة في autocomplete
- احتمالية أخطاء runtime

**الحل**:
إضافة توقيع الدالة للـ interface:

```typescript
interface MapState {
  // ... existing properties
  markTrainingComplete: (nodeId: string) => void;
}
```

### 3. إضافة Error Boundary لحماية التطبيق

**المشكلة**: التطبيق لا يحتوي على Error Boundary، مما يعني:

- أي خطأ في أي مكون يعطّل التطبيق بالكامل
- المستخدم يشوف شاشة بيضاء بدون رسالة واضحة
- لا يوجد آلية لتسجيل الأخطاء أو استعادة الحالة

**الحل**:

- إنشاء مكون `ErrorBoundary.tsx` في `[src/components/](src/components/)`
- استخدام React class component مع `componentDidCatch`
- عرض رسالة عربية واضحة للمستخدم
- إضافة زر "إعادة المحاولة" و "مسح البيانات"
- تغليف التطبيق الرئيسي في `[src/main.tsx](src/main.tsx)` بالـ Error Boundary

**الرسالة المقترحة للمستخدم**:

```
حصل خطأ غير متوقع 😔
ممكن تجرب تحدّث الصفحة أو تمسح البيانات وتبدأ من جديد
```
