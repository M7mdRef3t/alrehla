# 🌌 الرحلة — UX Blueprint
### Lead Designer: Apple HIG · Cognitive OS Edition

---

> **المنطلق الفلسفي للتصميم**
> الرحلة ليست تطبيق "مزاج". إنها مرآة حازمة ذكية. كل شاشة يجب أن تُجيب على سؤال واحد: *"ما القرار الذي يُساعد هذا الإنسان على اتخاذه الآن؟"*

---

## 🗺️ الهرمية المعلوماتية (Information Architecture)

```
الرحلة
├── 0. الملاذ (Sanctuary) ← حالة الطوارئ فقط
├── 1. خريطة المدارات (Orbit Map) ← الشاشة الرئيسية
│   ├── إضافة شخص (Quick Add Flow)
│   └── بطاقة الشخص (Node Detail)
├── 2. قراءة المدار (Orbit Reading) ← بعد كل تقييم
├── 3. المهمة (Mission Screen)
├── 4. نبض اليوم (Pulse Check)
├── 5. المسار (Trajectory Dashboard)
├── 6. المكتبة (Educational Library)
└── 7. الإعدادات (Settings / Profile)
```

---

## 🧭 نمط التنقل (Navigation Pattern)

**Tab Bar — 5 عناصر دائمة (iPhone)**

| 🏠 | 📊 | ⚡ | 📚 | 👤 |
|---|---|---|---|---|
| الخريطة | المسار | النبض | المكتبة | أنا |

- **لا Hamburger Menu** — التعقيد مخفي في Sheets لا في Drawers.
- **Bottom Tab** مثبّت دائماً — لا يختفي عند التمرير.
- **حالة Sanctuary**: يختفي Tab Bar بالكامل، يُعوَّض بـ Breathing Indicator.

**Gestures:**
- `Swipe Right from Edge` → رجوع
- `Long Press على Node` → Quick Actions Menu (حذف / تعديل / استعادة السيادة)
- `Pinch على الخريطة` → Zoom In/Out
- `Swipe Up على Node Card` → تفاصيل كاملة

## 📐 نظام التصميم

### الشبكة (Grid)
- Margin: **20pt** (iPhone 14)
- Column: **4 columns**, Gutter: **16pt**
- Safe Area: دائماً محترمة
- Min Touch Target: **44×44pt** (Apple HIG Mandatory)

### الطباعة (Typography)
| المستوى | الخط | الحجم | الوزن |
|---|---|---|---|
| Display | Almarai | 34pt | Black 900 |
| Title 1 | Almarai | 28pt | Bold 700 |
| Title 2 | Almarai | 22pt | Bold |
| Body | IBM Plex Sans Arabic | 17pt | Regular |
| Caption | IBM Plex Sans Arabic | 13pt | Medium |
| Micro | IBM Plex Sans Arabic | 11pt | SemiBold |

**Dynamic Type:** كل النصوص تدعم Accessibility Sizes حتى AX5.

### الألوان
| Token | Value | الاستخدام |
|---|---|---|
| `--space-void` | `#0a0e1f` | الخلفية العامة |
| `--soft-teal` | `#2dd4bf` | الإجراء الأساسي، المدار الأخضر |
| `--warm-amber` | `#f5a623` | تنبيهات الوعي، المدار الأصفر |
| `--ring-danger` | `#f87171` | طوارئ، مدار أحمر |
| `--text-primary` | `#f1f5f9` | النص الرئيسي |
| `--glass-bg` | `rgba(15,23,42,0.6)` | Cards |

**ملاحظة إمكانية الوصول:** نسبة تباين 4.5:1 مطلوبة. نسبة تباين 7:1 لـ Micro text.

## 📱 الشاشات الثماني

### 0 · ملاذ الطوارئ (Sanctuary Mode)

**المحفّز:** اكتشاف نمط استنزاف حاد أو ضغط `زر الطوارئ` أو استجابة لأسئلة Pulse Check.

#### Wireframe
```
┌─────────────────────────────┐
│                             │
│   ∿∿∿ حركة تنفس ∿∿∿       │
│  [Circle expanding slowly]  │
│                             │
│       وقّف كل شيء           │  ← 32pt Bold, centered
│   ركّز على نَفَس واحد        │  ← 17pt, muted
│                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━  │  ← animated progress
│                             │
│  [ جهة اتصال دعم ] ← Ghost │
│  [ غرفة التنفس    ] ← Ghost │
│                             │
└─────────────────────────────┘
```

#### المكوّنات
- **Breathing Orb**: SVG دائرة تتمدد وتتقلص على إيقاع 4-7-8. لا نص، لا تشتيت.
- **Haptic Feedback**: `UIImpactFeedbackGenerator(.light)` مع كل دورة تنفس.
- **Ghost Buttons** — أبيض 30% opacity — لا يجب أن تكون هي نقطة التركيز.
- **خط مساعدة**: يظهر بعد 30 ثانية فقط إذا لم يتفاعل المستخدم.

#### الحالات
| الحالة | السلوك |
|---|---|
| Loading | الـ Orb يبدأ صغيراً ويتوسع (Motion) |
| Active | دورة تنفس بلا انقطاع |
| Error (لا تحدث) | هذه الشاشة لا تفشل |

#### تفاعلات وإيماءات
- `Tap anywhere outside buttons` → لا يحدث شيء (عمدًا — لا إشتات)
- `Swipe Down` → Exit مع تأكيد "هل أنت بخير الآن؟"
- **لا Tab Bar** في هذا الوضع.

#### ملاحظات المصمم 🖊️
> هذه الشاشة هي **عكس التصميم**. النجاح يُقاس بمقدار ما **لم** نضعه عليها. الFocused breathing animation هي الـ Hero. كل element إضافي يُقلّل من فعاليتها.

### 1 · خريطة المدارات (Orbit Map) — الشاشة الرئيسية

**الهدف:** رؤية حالة جميع العلاقات دفعة واحدة. القرار يُتخذ بالنظر لا بالقراءة.

#### Wireframe
```
┌─────────────────────────────┐
│ الرحلة          [+] [⚙]    │  ← Nav Bar
├─────────────────────────────┤
│                             │
│         [أنا]               │  ← Center Dot (24pt)
│        / │ \                │
│    🔴    🟡    🟢           │  ← Orbit Rings
│   Ahmed Sara  Omar          │  ← Node Dots
│       🟡  🔴               │
│     Nour  Maha              │
│                             │
│  السيادة الكلية: ████░ 73%  │  ← Progress Pill
│                             │
├─────────────────────────────┤
│  🏠    📊    ⚡    📚    👤  │  ← Tab Bar
└─────────────────────────────┘
```

#### المكوّنات

**Orbit Rings (Canvas Layer)**
- 3 دوائر مركزية (أخضر, أصفر, أحمر) بـ stroke لا fill.
- العلاقات: نقاط (Nodes) 48×48pt مع Avatar أو حرف أول.
- المسافة من المركز = درجة القرب العاطفي.
- اللون = حالة المدار الحالية.

**Node Chip**
```
[ 🟢 A ] ← 48pt circle
  Ahmed   ← 11pt Caption, max 1 line
```

**السيادة الكلية Pill**
```
[ السيادة: ████░░ 73% ]  ← Bottom center, pill shape
```
- اللون يتغير: أخضر > 70%, أصفر 40-70%, أحمر < 40%.

#### حالات الشاشة
| الحالة | المظهر |
|---|---|
| **Empty State** | دائرة "أنا" وحيدة + "أضف شخصاً مؤثراً في حياتك الآن" + زر [+ ابدأ] |
| **Loading** | Shimmer على Nodes (Skeleton) |
| **Error** | Toast بالأسفل "تعذّر التحميل. اسحب للتحديث." |
| **فل** | أكثر من 12 شخص: أصغر Nodes + زر "عرض الكل" |

#### تفاعلات
- `Tap Node` → Node Detail Sheet (ينزلق من الأسفل)
- `Long Press Node` → Context Menu: تعديل / أرشفة / طوارئ
- `Pinch` → Zoom (0.5x - 2x)
- `Drag Node` → إعادة ترتيب داخل نفس المدار
- `Tap [+]` → Quick Add Flow (Full Sheet)

#### VoiceOver
- Node: *"أحمد، مدار أحمر، اضغط مرتين للتفاصيل"*
- Rings: labeled "مدار الأمان", "مدار التحفظ", "مدار الحماية"

### 3 · تدفق إضافة شخص (Quick Add Flow)

**الهدف:** إضافة علاقة جديدة وتصنيفها في أقل من 90 ثانية.

#### الخطوات (Multi-step Sheet)
```
Step 1/3: من هو؟
┌─────────────────────────────┐
│ اختر صفته في حياتك          │
│                             │
│ [شريك] [أهل] [صديق]        │  ← Chip Grid (3 per row)
│ [زميل] [مشرف] [آخر+]       │
│                             │
│ الاسم (اختياري):            │
│ [ _______________  ]        │
│                             │
│       [ التالي → ]         │
└─────────────────────────────┘

Step 2/3: كيف تشعر معه؟
(FeelingCheck — 4 أسئلة محددة)

Step 3/3: نتيجة المدار
(ResultScreen — قراءة فورية)
```

#### قواعد UX
- **Timer**: يُعرض في الخطوة 2 *"٢ دقيقة كافية — الجواب الأول هو الأصح"*.
- **Progress Bar**: Segmented 3 خطوات، يُعبّر عن الوضع الحالي.
- **لا skip**: كل سؤال مطلوب — شرحه: "إجابتك هي التشخيص".

#### Error States
| الحالة | التصرف |
|---|---|
| لم يختر صفة | زر [التالي] معطّل + tooltip "اختر صفة أولاً" |
| شبكة منقطعة | Local cache يُكمل الخطوات ويُزامن لاحقاً |

### 4 · قراءة المدار (Orbit Reading / Result Screen)

**الغرض:** الحكم النهائي — ماذا يعني هذا الشخص لي؟ وما المسار؟

#### Wireframe
```
┌─────────────────────────────┐
│ ┌─────────────────────────┐ │
│ │ 🔴 علاقتك مع [سارة]   │ │  ← Ring Badge
│ │ في مدار الحماية         │ │
│ └─────────────────────────┘ │
│                             │
│ [العنوان الرئيسي]           │  ← 28pt Bold
│ "الجلاد اللطيف"             │
│                             │
│ [Card: ماذا يحدث؟]          │
│ [Card: الأدوات المطلوبة]    │
│ [Card: خطة الخطوة ]         │
│                             │
│ [ ابدأ المسار الآن  ] ←Primary │
│ [ أضف للخريطة     ] ← Sec  │
└─────────────────────────────┘
```

#### المكوّنات
**Ring Status Banner**
```
[ 🔴 في مدار الحماية — سحب طاقة عالٍ ]
```
اللون الخلفي يعكس لون المدار (rose-50/80 للأحمر).

**Insight Cards** — Accordion قابل للتوسع:
1. "ماذا يحدث؟" — التشخيص النظري
2. "أدواتك" — ما تحتاجه لهذه الخطوة  
3. "التحديات المتوقعة" — العوائق والحلول
4. "خطة الخطوة" — ordered steps

**Share Card** (مخفية افتراضياً):
- Share كـ Image أو Text.

#### تفاعلات
- `Swipe Up` → يكشف المزيد من التفاصيل
- `Tap Card Header` → Expand/Collapse مع animation
- **Haptic**: `.notification(.success)` عند ظهور النتيجة

#### Empty / Error
- لا نتيجة (نادر): *"لم تكتمل البيانات — راجع الأسئلة"* + زر رجوع.

#### ملاحظات المصمم 🖊️
> **اللون هو الرسالة الأولى.** قبل أن يقرأ المستخدم أي نص، يجب أن يشعر بلون المدار. الـ Banner يأخذ 100% عرض الشاشة وخلفية ملوّنة كافية للحصول على هذا التأثير.

### 5 · شاشة المهمة (Mission Screen)

**الغرض:** تتبع خطوة محددة مع شخص محدد خلال فترة زمنية.

#### Wireframe
```
┌─────────────────────────────┐
│ ← مهمتك مع [سارة]          │  ← Nav: Back
├─────────────────────────────┤
│                             │
│ التقدم:  ██████░░ 3/5       │  ← Progress Arc
│                             │
│ ┌──────────────────────────┐│
│ │ ✅ الخطوة 1: لاحظت نبضتي│ │  ← Completed
│ │ ✅ الخطوة 2: كتبت للورقة │ │
│ │ ✅ الخطوة 3: تحدثت بهدوء │ │
│ │ ◻ الخطوة 4: قيّمت النتيجة│ │  ← Current
│ │ ◻ الخطوة 5: قررت         │ │  ← Pending
│ └──────────────────────────┘│
│                             │
│ [ أكملت هذه الخطوة ✓ ]     │  ← Primary CTA
│                             │
└─────────────────────────────┘
```

#### المكوّنات
- **Step List**: Ordered list مع checkbox (custom, لا system).
- **Completed Step**: strikethrough + emerald tint.
- **Current Step**: أكثر brightness + Right border accent.
- **Progress Arc**: Circular progress 64pt في الأعلى.

#### حالات
| الحالة | المظهر |
|---|---|
| غير مبدوأة | "هل أنت جاهز للبدء؟" + [ابدأ الآن] |
| مكتملة | Celebration Haptic + "أحسنت — المدار تحسّن" |
| قديمة (+7 أيام) | Badge "يحتاج مراجعة" على الـ Tab |

#### Accessibility
- كل Checkbox: label *"الخطوة 3 من 5: تحدثت بهدوء — اضغط مرتين للتأشير"*
- VoiceOver يقرأ التقدم: *"3 من 5 خطوات مكتملة"*

### 6 · نبض اليوم (Pulse Check Modal)

**الغرض:** تقييم يومي سريع (أقل من 2 دقيقة) لضبط المدارات تلقائياً.

#### Wireframe
```
┌─────────────────────────────┐
│ نبض اليوم · +-2 دقيقة       │
├─────────────────────────────┤
│                             │
│ "كيف حال طاقتك الآن؟"      │  ← Question
│                             │
│  😵 ← ────────■──── → 😊  │  ← Slider (Haptic)
│     مستنزف    7   ممتلئ    │
│                             │
│    ──────────────────       │
│                             │
│   التالي →                  │
└─────────────────────────────┘
```

#### قواعد التصميم
- المحفّز: Push Notification ساعة 8م + In-app Banner.
- **4 أسئلة فقط** بـ Slider or Chips, لا نصوص طويلة.
- **Skip مسموح** — مع رسالة "سجّل عندما تكون جاهزاً".
- إذا كانت النتيجة < 3/10: **يُفعّل Sanctuary Mode** مباشرة.

#### ملاحظات المصمم 🖊️
> Slider لا Stepper. اللمس العاطفي يعمل بشكل أفضل على Scale متصل. الـ Haptic عند القيم الحدية (1, 5, 10) يُعطي إحساساً بالثقل الحقيقي للاختيار.

## ♿ إمكانية الوصول الشاملة (Accessibility Checklist)

### WCAG 2.2 AA Requirements
| المعيار | التطبيق |
|---|---|
| 1.4.3 Contrast | نسبة تباين 4.5:1 لكل نص. أدوات: WCAG Contrast Checker |
| 1.4.4 Resize Text | Dynamic Type حتى AX5 — اختبر مع Xcode Accessibility Inspector |
| 2.4.3 Focus Order | ترتيب منطقي RTL: يمين ← يسار ← أسفل |
| 2.5.3 Motion | `@prefers-reduced-motion` — كل Animations يمكن إيقافها |
| 3.2.2 On Input | لا تغيير سياق تلقائي عند Slider دون تأكيد |

### VoiceOver (Arabic)
- كل Node يُعلَن: *"[الاسم]، [الصفة]، مدار [اللون]، اضغط مرتين للتفاصيل"*
- Breathing Orb: *"رسم دائري متحرك، دورة تنفس نشطة"*
- Progress bars: *"التقدم 60 بالمائة"*
- `accessibilityLabel` إلزامي على كل Icon-only button.

### Dynamic Type
- خطوط Vector فقط (لا صور نصية أبداً).
- Cards تتوسع رأسياً عند الأحجام الكبيرة — لا overflow خفي.
- Minimum size مضمون عند 11pt (Micro) حتى مع AX1.

| Load Node | Scale أو Fade-in من المركز | 300ms ease-out |
| Sanctuary Enter | Full-screen Crossfade | 500ms |
| Step Complete | Checkbox Bounce + Emerald Fill | 250ms spring |
| Ring Change | Color Crossfade (node) | 400ms |
| Error Toast | Slide-in from bottom | 200ms |
| Celebration | Subtle Confetti burst | 600ms |

**Spring Physics**: `stiffness: 300, damping: 25` للعناصر الحيوية.
**لا Motion** في حالة Sanctuary — ثبات تام ما عدا الـ Breathing Orb.

## 🔏 قواعد المنصة الخمسة (Design Principles)

1. **السيادة تُكتسب، لا تُعطى** — كل شاشة تنتهي بقرار ينتمي للمستخدم، لا للتطبيق.
2. **اللون هو اللغة الأولى** — الأحمر يُقلق قبل القراءة. الأخضر يُريح قبل الفهم.
3. **أقل عناصر = قرار أوضح** — في لحظات الضغط، نحذف لا نضيف.
4. **الكلمة مباشرة، لا وعظية** — "وقّف" لا "ربما تحتاج إلى حدود". "قرّر" لا "فكّر في...".
5. **الخريطة هي الذاكرة** — المستخدم لا يتذكر نصاً يقرأه. يتذكر نقطة حمراء رآها.

## 📝 ملاحظات المصمم الختامية

> **العلاقة مع المستخدم في لحظة الأزمة**
> الرحلة في حالة Sanctuary يجب أن تكون **أبطأ** من معدل التفكير المتسارع للمستخدم. إذا كان الـ UI سريعاً ومتفاعلاً، يزيد القلق. الـ Breathing Orb البطيء هو إعادة تعليم الجهاز العصبي. **هذا هو الـ Product.**

> **الفرق عن منافسي "الصحة النفسية"**
> Headspace و Calm يبيعان **السكون**. الرحلة تبيع **الوضوح**. الأول شعور، الثاني قرار. كل قرار تصميمي يجب أن يُسأل: *"هل هذا يجعل الوضع أوضح أم أكثر ارتياحاً؟"* — نحن نختار الوضوح.

> **RTL First**
> الواجهة مصمّمة للعربية أصلاً، لا مترجمة. الـ Tab Bar يُقرأ من اليمين. الـ Back button على اليمين. Flow منطقي كامل للعين العربية.
