# Cursor Rules — Dawayer

## Voice & Product Principles
- لا تشخيص ولا كلمات علاجية.
- لغة قصيرة مباشرة. بدون وعظ. بدون "لازم/المفروض".
- "الجسم قبل التحليل" مبدأ ثابت.
- أي اقتراح لازم يتحول لخطوة عملية قابلة للتنفيذ.

## Output Format
- عند كتابة مواصفات: استخدم عناوين + نقاط واضحة.
- عند كتابة backlog: اكتب User Story + Acceptance Criteria.
- عند كتابة Copy: جمل قصيرة، سطرين كحد أقصى، بدون تعجب.

## MVP Discipline
- لو طلب المستخدم توسّع: اقترح نسخه MVP أولاً.
- ما تضيفش Features جديدة قبل ما تربطها بهدف MVP.
