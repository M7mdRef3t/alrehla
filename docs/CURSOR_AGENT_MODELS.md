## توصيف كل موديل وأفضل استخدام عملي

### Claude 4.5 Opus

- **التوصيف:** أقوى موديل Anthropic في Cursor. تفكير عميق، ردود دقيقة، يفهم سياق معقد.
- **الواقع العملي:** أغلى موديل (خصوصاً الـ Output). كوتا الطلبات أو الرصيد بيخلص منه بسرعة لو استخدمته لكل حاجة.
- **أفضل استخدام:** مهام صعبة مرة واحدة: تصميم معماري، refactor كبير، تحليل باجات معقدة، قرارات تقنية محتاجة تفكير طويل. **لا** تستخدمه للطلبات البسيطة (سطر، تعديل بسيط، أوامر تيرمنال).

---

### Claude 4.5 Sonnet

- **التوصيف:** توازن بين القوة والتكلفة. Agent + Thinking، ويدعم Max Mode حتى 1M توكن.
- **الواقع العملي:** أنسب للشغل اليومي لو عايز جودة عالية ومش عايز تدفع Opus. نافذة 1M مفيدة لمشاريع كبيرة أو محادثات طويلة.
- **أفضل استخدام:** التطوير اليومي، كتابة features متوسطة التعقيد، مراجعة كود، إصلاح باجات، شات طويل على نفس المشروع. اختيار عملي للـ "default قوي".

---

### Composer 1

- **التوصيف:** موديل Cursor نفسه (مِلكهم). مُحسّن للكود والـ Agent (ملفات، أوامر، تنفيذ).
- **الواقع العملي:** أرخص من Claude/GPT، وسريع. مفيش Max Mode — السياق 200k.
- **أفضل استخدام:** كتابة وتعديل كود، تنفيذ أوامر، فتح ملفات متعددة، مهام Agent واضحة (مثلاً "املا هذا الملف"، "شغّل الاختبارات"). مش الأنسب لو محتاج تفكير رياضي/منطقي معقد جداً — هنا Claude/GPT Thinking أفضل.

---

### Gemini 3 Flash

- **التوصيف:** موديل Google خفيف وسريع. Agent + Thinking، ورخيص جداً.
- **الواقع العملي:** أرخص خيار مع قدرة Thinking. جودة أقل شوية من Pro/Claude في المهام المعقدة، لكن كافية لمعظم الطلبات اليومية.
- **أفضل استخدام:** طلبات كثيرة وخفيفة: شرح كود، إصلاحات بسيطة، أوامر صغيرة، تجربة أفكار سريعة. لو الكوتا أو الرصيد محدود — اختاره كـ default وادّخر Pro/Claude للمهام الصعبة.

---

### Gemini 3 Pro

- **التوصيف:** موديل Google الأقوى. Agent + Thinking، ويدعم Max Mode حتى 1M.
- **الواقع العملي:** أغلى من Flash، أقل تكلفة من Claude Opus. جودة عالية ومناسب لمشاريع كبيرة السياق.
- **أفضل استخدام:** مهام تحتاج جودة عالية وسياق كبير: تحليل ملفات كثيرة، خطط refactor، شات طويل على codebase. بديل عملي لـ Claude Sonnet لو prefer Google.

---

### GPT-5.2

- **التوصيف:** موديل OpenAI متقدم. Agent + Thinking، سياق 272k.
- **الواقع العملي:** توازن جيد بين السعر والأداء. معروف بفهم التعليمات والكود.
- **أفضل استخدام:** تطوير عام، features متوسطة، إصلاح باجات، أوامر واضحة. اختيار مريح لو اعتدت على أداء GPT.

---

### GPT-5.2 Codex

- **التوصيف:** نفس عائلة 5.2 لكن مُحسّن للكود (اسم Codex).
- **الواقع العملي:** نفس السعر تقريباً مع 5.2، مع تركيز أفضل على توليد/تعديل كود.
- **أفضل استخدام:** كل ما هو كتابة أو تعديل كود، اقتراح دوال، تحسين أداء، اتباع patterns. لو شغلك كله كود — اختر Codex على 5.2 العادي.

---

### Grok Code

- **التوصيف:** موديل xAI للكود. Agent + Thinking، وأرخص جداً في الجدول.
- **الواقع العملي:** التكلفة منخفضة جداً — مناسب لاستهلاك كبير أو تجارب.
- **أفضل استخدام:** طلبات كثيرة وخفيفة، تجربة أوامر، توليد كود بسيط، أو لما تريد توفير الرصيد. الجودة قد تكون أقل قليلاً من Claude/GPT في المهام المعقدة — جربه وقرر حسب مشروعك.

## Auto (الاختيار التلقائي)

في Cursor تقدر تفعّل **Auto** عشان يختار الموديل المناسب للمهمة والموثوقية حسب الحمل. عملياً: لو مش حابب تفكر "أي موديل دلوقتي"، اترك Auto. لو عندك رصيد محدود أو تفضيل معين، اختر موديل ثابت (مثلاً Gemini 3 Flash أو Composer 1).

---

## Max Mode

لو المشروع كبير أو المحادثة طويلة، تفعيل **Max Mode** يوسّع نافذة السياق (مثلاً 1M لـ Sonnet و Gemini 3). بيكون أبطأ وأغلى — استخدمه فقط لما تحتاج فعلاً سياق أكبر من 200k.

## إعدادات موصى بها — أقوى نتيجة بأقل التكاليف

الهدف: كل موديل في مكانه، وتقليل التكلفة مع الحفاظ على جودة النتيجة. التطبيق **يدوي** من واجهة Cursor (القائمة تحت صندوق الـ Chat/Composer).

### 1. الموديل الافتراضي (Chat / الشات العام)

| الإعداد | القيمة الموصى بها | السبب |
|---------|-------------------|--------|
| **Default model** | **Gemini 3 Flash** | أرخص موديل مع Agent + Thinking، كافي لمعظم الأسئلة والطلبات اليومية. |
| بديل | Composer 1 | لو تفضّل موديل Cursor للكود؛ رخيص وسريع. |
| بديل للجودة الأعلى | Claude 4.5 Sonnet | لو الرصيد يسمح وتفضل جودة أعلى على الافتراضي. |

**كيف:** من نافذة Chat اختر الموديل من القائمة تحت صندوق الكتابة. الموديل اللي تختاره يبقى هو المستخدم في الجلسة حتى تغيّره.

---

### 2. Composer / Agent (كتابة كود وتنفيذ أوامر)

| الإعداد | القيمة الموصى بها | السبب |
|---------|-------------------|--------|
| **Model في Composer** | **Composer 1** أو **GPT-5.2 Codex** | مُحسّنان للكود؛ Composer أرخص، Codex جودة عالية. |
| بديل رخيص | Gemini 3 Flash أو Grok Code | استهلاك كثير بأقل تكلفة. |

---

### 3. متى تبدّل الموديل يدوياً (كل موديل في مكانه)

| نوع المهمة | الموديل الموصى به |
|------------|-------------------|
| سطر، تعديل بسيط، شرح سريع | الإبقاء على Flash أو Composer — لا داعي لـ Pro/Opus. |
| Feature متوسطة، إصلاح باج، مراجعة كود | Flash أو Composer أو Sonnet حسب الرصيد. |
| تصميم معماري، refactor كبير، تحليل باجات معقدة | **التبديل لـ Claude 4.5 Sonnet** أو **Gemini 3 Pro**. |
| قرار تقني صعب، خطة طويلة، تفكير عميق | **التبديل لـ Claude 4.5 Opus** (مرة واحدة ثم الرجوع للافتراضي). |
| مشروع كبير، محادثة طويلة، سياق أكبر من 200k | تفعيل **Max Mode** مع Sonnet أو Gemini 3 Pro. |

---

### 4. Auto vs موديل ثابت

| الخيار | متى تختاره |
|--------|-------------|
| **Auto** | لو مش حابب تفكر في الموديل؛ Cursor يختار حسب المهمة والحمولة. قد يزيد الاستهلاك. |
| **موديل ثابت (مثلاً Gemini 3 Flash)** | لو عايز تحكم في التكلفة وتضع "كل موديل في مكانه" يدوياً — اختر الافتراضي رخيص وبدّل لـ Sonnet/Pro/Opus عند الحاجة فقط. |

---

### 5. ملخص خطوات التطبيق

1. **Cursor Settings** (أو من واجهة Chat/Composer): اختر **Default model = Gemini 3 Flash** (أو Composer 1).
2. في **Composer**: اختر **Composer 1** أو **GPT-5.2 Codex** لكتابة الكود.
3. عند طلب **معقد** (معمارية، refactor كبير، تحليل): غيّر الموديل من القائمة إلى **Claude 4.5 Sonnet** أو **Gemini 3 Pro** قبل الإرسال.
4. عند **أصعب مهمة مرة واحدة**: اختر **Claude 4.5 Opus** ثم ارجع للافتراضي بعد الانتهاء.
5. **Max Mode**: فعّله فقط عند الحاجة لسياق أكبر من 200k.

قاعدة المشروع في `.cursor/rules/model-and-cost.mdc` توجّه المساعد ليكون مختصراً في الطلبات البسيطة ويقترح التبديل لموديل أقوى عند المهام المعقدة.
