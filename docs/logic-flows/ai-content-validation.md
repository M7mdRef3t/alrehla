# AI Content Validation Logic Flow

## Goal
The goal of this feature is to ensure that any content generated by the AI agent autonomously (like daily questions or recovery scripts) aligns strictly with the core therapeutic principles of "Dawayir" before it is presented to the user.

## Mental Model
- The AI Agent generates a response or suggestion.
- The `DecisionEngine` intercepts this response and extracts the textual content.
- The system evaluates the content against `CORE_PRINCIPLES.ts`.
- If it violates any principles (e.g., toxic positivity, guru speak), it is rejected.
- If it passes, it is allowed to proceed.

## Inputs / Outputs
- Inputs: An `AIDecision` object containing a `payload` with generated content.
- Outputs: A boolean `passed` flag, and an optional `reason` string if validation fails.

## States
- `evaluating`: The engine is actively extracting and checking text.
- `passed`: The content aligns with core principles.
- `rejected`: The content violates core principles.

## Transitions
1. `evaluating -> passed` when `isAlignedWithPrinciples` returns true.
2. `evaluating -> rejected` when `isAlignedWithPrinciples` returns false.

## Edge Cases
- Unknown payload structure: The system safely falls back to returning true if it cannot extract text, or handles specific fields like `message`, `text`, `doSay`/`dontSay`.
- Empty text string: Passed by default.

## Failure & Fallback
- If the text violates principles, the decision engine prevents execution and logs the reason.

## Performance Constraints
- Target complexity: O(N) where N is the length of the string, as it uses basic string matching.
- Max latency: < 5ms.

## Security Constraints
- Validation rules: Strictly enforced via `CORE_PRINCIPLES.ts` matching.
- Authorization boundary: Evaluated completely on the local client side.

## Acceptance Criteria
1. The engine successfully extracts text from common payload formats (`generatedQuestion`, `greeting`, `missionDescription`, etc.).
2. The engine uses `isAlignedWithPrinciples` to check for violations.
3. Content that fails returns `{ passed: false, reason: "..." }`.
