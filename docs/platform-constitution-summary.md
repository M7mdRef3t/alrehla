# Platform Constitution Summary

## Identity
- Product name: Al-Rehla.
- Core promise: help users identify draining relationships, reclaim focus, and apply practical daily actions.

## Core Model
- Relationship map with zones: red, yellow, green, grey.
- Tactical outputs: state, goal, promise, daily mission, execution plan, expected traps.
- AI is core, but every AI path must have a stable fallback template.

## Release Waves
- Wave 1 (Awakening): map + basic diagnosis + pulse + daily mission + text AI.
- Wave 2 (Control): self-regulation tools and adaptive/GenUI assistance.
- Wave 3 (Roots): family tree patterns and relationship atlas insights.

## Role Model
- Regular users: only current wave features.
- Developer/Admin roles: full platform visibility and operation.

## Delivery Requirements
- Feature flags control visibility, not code existence.
- Locked features must be discoverable and graceful (Coming Soon), not broken.
- Data schema for future features must exist from day one.

## AI Contract
- AI must respect feature availability.
- AI should redirect locked requests to currently available tools.
- UI/content should clearly signal whether output came from AI or fallback template.
